# 팀 일정 관리 툴 기획안 (초안)

---

## 1. 프로젝트 개요

| 항목       | 내용                                                                               |
| ---------- | ---------------------------------------------------------------------------------- |
| 프로젝트명 | 팀 일정 관리 툴 (내부 명칭 미정)                                                   |
| 목적       | 9인 팀의 업무 일정과 현황을 관리하고, Slack과 연동하여 알림 및 빠른 일정 등록 지원 |
| 사용자     | 찾아줘! 팀 9명                                                                     |
| 기술 스택  | Next.js 14, TypeScript, Supabase, Tailwind CSS, Slack API                          |
| 배포       | Vercel (프론트엔드), Supabase (DB/백엔드)                                          |

---

## 2. 목적 및 배경

> "팀 내 업무 일정을 서로 메시지로 공유하다 보니 놓치는 경우가 발생하여,
> 한눈에 확인할 수 있는 공용 일정 관리 툴이 필요하다"

- 별도 도메인이나 앱 없이 1인 개발로 제작
- 팀 내부 전용 툴로, 외부 공개 불필요
- 포트폴리오 겸 실습용 목적

---

## 3. 사용자 및 권한

MVP 단계에서는 권한을 단순하게 가져갑니다.

| 역할                        | 권한                           |
| --------------------------- | ------------------------------ |
| 팀원 (로그인한 모든 사용자) | 일정 조회 / 등록 / 수정 / 삭제 |

추후 팀장 역할을 분리하여 삭제 권한을 제한하는 방향으로 확장 가능.

## 4. 기능 목록

### MVP (1차 개발)

| 기능              | 설명                                           |
| ----------------- | ---------------------------------------------- |
| 로그인 / 로그아웃 | Supabase Auth 기반 이메일 로그인               |
| 일정 등록         | 제목, 담당자, 마감일, 상태 입력                |
| 일정 조회         | 칸반 보드 형태로 상태별 분류 표시              |
| 일정 수정         | 제목, 담당자, 마감일, 상태 변경                |
| 일정 삭제         | 삭제 버튼으로 완전 삭제 (하드 딜리트)          |
| 상태 커스터마이징 | 팀원이 직접 상태 추가 / 수정 / 삭제 / 순서변경 |
| Slack 알림        | 새 일정 등록 시 팀 채널로 알림                 |

### 2차 개발 (MVP 이후)

| 기능               | 설명                                                  |
| ------------------ | ----------------------------------------------------- |
| Slack 내 일정 추가 | 봇 DM에서 `일정추가 필드브레인워크` 입력 시 바로 등록 |
| 담당자 필터        | 칸반 보드에서 담당자별 필터링                         |
| 실시간 동기화      | Supabase Realtime으로 다른 팀원 변경사항 즉시 반영    |

### 추후 고려

| 기능              | 설명                                 |
| ----------------- | ------------------------------------ |
| 반복 일정         | 매주 반복되는 업무 자동 생성         |
| 통계 / 대시보드   | 완료율, 담당자별 업무량 시각화       |
| 양방향 Slack 연동 | Slack에서 일정 수정 / 상태 변경 가능 |

## 6. 데이터 구조 (DB 설계)

### profiles 테이블

팀원 정보 (Supabase Auth 연동)

| 컬럼       | 타입        | 설명                 |
| ---------- | ----------- | -------------------- |
| id         | uuid        | auth.users 참조 (PK) |
| name       | text        | 팀원 이름            |
| created_at | timestamptz | 생성일               |

### task_statuses 테이블

커스터마이징 가능한 상태 목록

| 컬럼        | 타입        | 설명                         |
| ----------- | ----------- | ---------------------------- |
| id          | uuid        | PK                           |
| name        | text        | 상태 이름 (예: 진행 중)      |
| color       | text        | 칸반 컬럼 색상 (예: #FF5733) |
| order_index | int         | 칸반 컬럼 순서               |
| created_at  | timestamptz | 생성일                       |

### tasks 테이블

일정 데이터

| 컬럼        | 타입        | 설명                   |
| ----------- | ----------- | ---------------------- |
| id          | uuid        | PK                     |
| title       | text        | 일정 제목              |
| description | text        | 상세 내용 (선택)       |
| status_id   | uuid        | task_statuses 참조     |
| assignee_id | uuid        | profiles 참조 (담당자) |
| due_date    | date        | 마감일                 |
| created_by  | uuid        | profiles 참조 (작성자) |
| created_at  | timestamptz | 생성일                 |
| updated_at  | timestamptz | 수정일 (자동 업데이트) |

## 7. Slack 연동 구조

```
[단방향 알림 - MVP]
새 일정 등록 (Supabase INSERT)
→ Database Webhook 트리거
→ Edge Function 실행
→ Slack 팀 채널에 알림 메시지 전송

[양방향 연동 - 2차]
팀원이 봇 DM에서 "일정추가 필드브레인워크" 입력
→ Slack Bot이 메시지 감지
→ Edge Function 호출
→ tasks 테이블에 INSERT
→ 봇이 DM으로 "✅ 일정이 추가되었어요!" 응답
→ 팀 채널에 알림
```
